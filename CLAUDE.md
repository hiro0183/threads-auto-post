# CLAUDE.md — コンサル垢 日次投稿の実行ルールブック

このリポジトリは **治療院経営コンサル垢（ヒロ先生）専用**。
ラポール整体院（threads_tool_rapport / @rapport.sango）とは別物。コードもテーマも口調も絶対に混同しない。

**このファイルは日次実行者（Sonnet・claude_headless / またはクラウドルーティン）向けの完全ルール。**
ここに書いてないことは勝手に判断せず、Obsidian「⭐運用司令室」体制に従いエスカレーションする。
ルールブック（このファイル等）の改定権は月1回のスキーム見直し（Fable 5・手動起動）＋ヒロさん承認のみ。日次セッションでこのファイルを書き換えない。

> **費用の絶対原則（2026-07-09〜）:** 生成・品質ゲートは **サブスク完結・API課金ゼロ**。
> `content_generator.py` の `claude_headless` は `ANTHROPIC_API_KEY` を環境から除外して `claude -p` を回す。
> 直接 Anthropic API を叩く経路（旧 generate_day.py の非planモード等）は**予備手段**。日次では使わない。

## 毎朝の手順（この順番で必ず・朝5時完了が原則）

1. **リポジトリ確認:** ここが `threads_tool`（コンサル垢）であることを確認（taboo.md #8。rapport側との取り違え防止）
2. **週間プランを読む:** `posts/weekly_plan/*.json` から翌日分を含むプランを探す（`generate_day.find_weekly_plan_for_date` と同じ探索）
   - **その日を含む weekly_plan が無ければ、何もせず終了**（Opus 4.8 の週次セッション未実施日はエラーにしない）
   - プランの `directives`（改善指示）があれば必ず反映する
3. **重複チェックの準備:** 過去7日分の使用済みキャッチを読む（`load_used_catches(days=7)` 相当。同一フックの7日以内再使用は禁止）
4. **翌日分の本文を書く（planモード）:** weekly_plan の各 `day_entry`（slot / hook / type / theme / conclusion / cta）について、
   - **フックは一字も変えない**（Opus 4.8 が週次で確定済み。フックの改変は憲法違反）
   - 本文（2〜3投稿目）だけを下の「投稿の書き方」に従って執筆する
   - `posts/{date}.json` に slot をキーにして保存（値＝投稿文の配列）
5. **品質ゲート（独立ステップ）:** 完成した本文だけを渡して Haiku に判定させる（`prompts/gate.md` の6項目）。
   - 生成側の会話・意図は渡さない（思い込みの引き継ぎ防止）
   - NG → 本文のみ再生成（フックは変えない）。**同一スロット2回NGで自動修正を諦め、人間確認へ回す**（無限ループ防止）
6. **プレビュー・司令室更新:** `export_preview` → `ops_dashboard` を最新化
7. **push:** `git add posts/ && git commit && git push`（Renderのscheduler.pyが自動で拾って時刻通り投稿）

## posts/{date}.json のフォーマット

- キーは weekly_plan のその日の `day_entries` にある slot
- 値は「投稿文の配列」（ツリー ＝ 2〜3要素）。**単体（1要素）は禁止**（taboo.md #3）
- スロットを休止する場合は `[]`（空配列）
- CTA付きスロットは weekly_plan の `cta` に従う（**1日 2〜3本まで**。MAX_CTA_PER_DAY=3）

## 投稿の書き方（フック固定・本文のみ執筆）

**1投稿目（フック）: weekly_plan の `hook` をそのまま使う（一字も変えない）**

**2投稿目以降（本文）: フックが約束した内容を回収する**
- フックと本文が矛盾・すり替わらないこと（gate.md #1）
- 誰向けかを一人に絞る（読者像＝月商100万前後の一人治療院経営者。gate.md #2）
- 保存・コメントしたくなる中身（誰向け／何の問題／保存理由／行動導線）
- CTA付きなら最終投稿にコメント誘導 or LINE導線を自然に組み込む（単体では出さない）

**文体の絶対ルール:**
- **丁寧語で統一**（〜ません／お客様／〜ではありません）
- **一人称は「僕」**
- 締めは励まし系（「一緒に頑張りましょう」「あなたなら大丈夫です」）
- **架空実績・未確認数字・盛った成果は禁止。使ってよい実数は `prompts/persona.md` のペルソナ実数のみ**

## 禁止事項（憲法＝taboo.md。違反したら即修正）

1. **過去の伸びた投稿の全文を読んで書き換え・リミックスしない**（平均化でホームランが消える実証あり）。参考にするのは型ラベルと構造まで
2. **重複防止は直近7日まで**（30日重複防止は伸びる型まで殺す）
3. **単体投稿を作らない。ツリー（2〜3投稿）一択**
4. **「実は〇〇」書き出し禁止**（文中も避ける）
5. **CTA付きは1日2〜3本まで**（CTA過多はリーチ壊滅の実測あり）
6. **架空実績・未確認数字・盛った成果は禁止**（実数は persona.md のみ）
7. **同じ型・語尾の連続使用に注意（鮮度管理）**（同一語尾の使いすぎで95%ダウンの実測あり）
8. **実行前に必ずリポジトリを確認**（threads_tool＝コンサル垢。rapport側と混同禁止）

## 品質ゲートの判定項目（gate.md・独立ステップ）

1. フックの約束を本文が回収しているか
2. 誰向けかが一人に絞れているか（persona.md の読者像）
3. persona.md のペルソナ実数**以外**の数字・実績が無いか
4. taboo.md の禁じ手8つに抵触していないか
5. 丁寧語・一人称「僕」で統一されているか
6. 直近7日と型・フック・テーマが重複していないか

## エスカレーション（自分で直さず上に上げる）

- その日の weekly_plan が無い → 何もせず終了（Opus 4.8 週次セッション待ち）
- 品質ゲート2回連続NG → そのスロットは要人間確認に回す（無理に出さない）
- ルールがおかしいと感じた → **変えずに** 実行ログにメモ（週次セッションが拾う）

## 運用メモ

- **投稿実行はRender（scheduler.py）が担当。** このセッションでは投稿APIを直接叩かない
- **トークンは DB永続化済み**（`db_state.threads_token` / `token_manager` がDB優先）。再デプロイで巻き戻らない
  - 手動再認証した直後だけ `python token_manager.py --seed` で新トークンをDBへ発行する
- トークン期限が7日以内に迫っていたらヒロさんに再認証を依頼（`python threads_auth.py`）
- 生成・ゲートは**サブスク完結**。`ANTHROPIC_API_KEY` を環境に残さない（残すとAPI課金側で動く）
- 週次企画（フック確定）は **Opus 4.8**（毎週月曜05:10・`run_weekly_session.ps1`）、日次の本文執筆は Sonnet、品質ゲートは Haiku。全部 claude_headless（サブスク・API課金ゼロ）
- **Fable 5 は月1回のスキーム見直し（手動）＋品質劣化時のフォールバックのみ**（ホームランが2週連続下落したら `--model opus`→`--model fable` に戻す）
