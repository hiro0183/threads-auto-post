# 品質ゲート（独立ステップ専用・quality_gate.py用）

> 出典: 指示書 §7。**本文生成とは別のAPI呼び出しで実行する。** 生成側の会話履歴・プロンプトは渡さず、完成した投稿本文だけを渡して判定させる（生成時の思い込みを引き継がせないため）。

## 判定項目（1つでもNGなら本文のみ再生成・フックは変更しない）

1. フックが約束した内容を本文が回収しているか（フックと本文が矛盾・すり替わっていないか）
2. 誰向けかが一人に絞れているか（`persona.md`の読者像からズレていないか）
3. `persona.md`のペルソナ実数**以外**の数字・実績が使われていないか
4. `taboo.md`の禁じ手8つに抵触していないか（特に：リミックス由来の表現／単体投稿／CTA本数超過／「実は〇〇」書き出し）
5. 丁寧語・一人称「僕」で統一されているか
6. 直近7日の投稿と型・フック・テーマが重複していないか（機械検証と重複してもよい二重チェック）
7. **【エンゲージ床・2026-07-12追加】この投稿に「いいね／保存／リプする具体的理由」が最低1つあるか。** 次のいずれも無ければNG＝本文再生成:
   - 保存動機: 真似できる手順・基準値・チェック項目のような具体（`body_rules.md` #9）
   - リプ動機: 即答できる問い・二択・当事者性のある呼びかけ（締めが開いているか。#10）
   - いいね動機: 具体的な場面・セリフ・本音（#11）
   - **抽象的な経営説教・一般論・応援だけで閉じている投稿はNG**（実データで反応ゼロの主因）。「◯◯は△△です」の結論連打で反応の余地が無いものは落とす。

## 補助レンズ（小川脳みそ`03_テンプレ_SNS投稿採点.md`/`04_テンプレ_コピー採点.md`より一般化して援用。上記6項目を補強する参考軸であり、これ自体が合否を決めるものではない）

- **コピー5軸**: 具体性／便益／根拠／反論処理／独自性
- **SNS投稿4軸**: 誰向け／何の問題／保存理由（なぜ保存・コメントしたくなるか）／行動導線

## 出力フォーマット

「OK」または「NG＋理由＋修正指示」の構造化フォーマットにし、`quality_gate.py`がNG時に本文再生成をトリガーできるようにする。

**2回連続でNGが出た投稿は自動修正を諦めて人間のプレビュー確認に回す**（無限ループ防止）。
